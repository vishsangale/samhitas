# Thread 5: Learned / adaptive equivariance

**Math source:** Lie group theory, representation theory (irreducible representations,
group generators/Lie algebra), equivariant deep learning.

## Motivation

Equivariant architectures (CNNs for translation, spherical CNNs for rotation, etc.) hard-
code a *known* symmetry group into the layer. This is a strong, useful prior when the
symmetry is known and exact — but it is also exactly the kind of hand-supplied domain
knowledge the bitter lesson warns will be outcompeted by scale once the model can afford to
learn the pattern itself. The interesting mathematical question is whether the *machinery*
of equivariance (build a layer that commutes with a group action) can be kept while making
the group itself a learned object — parameterize a small set of Lie algebra generators and
let gradient descent discover which continuous symmetry the data actually has, rather than
a human picking it in advance.

## Architectural hypothesis

A layer parameterized by learnable Lie algebra generators (with the layer constructed to
be exactly equivariant to the one-parameter subgroups those generators produce, e.g. via
the matrix exponential) can, on synthetic data generated by an actual unknown symmetry
group action, recover generators close to the true group's generators purely from the
training objective, and once converged, match the sample efficiency of a hand-built
equivariant network for that same group — without a human ever specifying the group.

## Falsifiable prediction

On synthetic data explicitly generated by applying a known Lie group action (e.g., SO(2)
rotations, or a scaling group) to base examples, the learned-generator layer's generators
should converge (up to basis change/normalization) to the true infinitesimal generators of
that group, measurable via a generator-alignment metric, and the model's sample-efficiency
curve (accuracy vs. training set size) should approach — not necessarily match exactly,
but approach within a small, pre-registered margin — a hand-built equivariant baseline for
the same group. If generators don't converge to anything resembling the true group, or if
sample efficiency doesn't approach the hand-built baseline even after generator
convergence, the thread is falsified. This is the one thread in the portfolio with the
most "bitter lesson trap" risk: if a plain, larger MLP with more data closes the gap just
as well without ever discovering symmetry explicitly, that is a valid and important
falsification of the *architecture's* value-add, even if the generator-recovery part works.

## Minimal experiment

- Synthetic dataset generator: apply a chosen 1- or 2-parameter Lie group to a small set of
  base patterns/images, label by an invariant of the base pattern.
- Train (a) learned-generator equivariant layer, (b) hand-built equivariant layer for the
  true group, (c) plain MLP/CNN baseline with matched compute, across a sweep of training
  set sizes.
- Measure: generator alignment metric over training; sample-efficiency curves for all
  three; explicitly report whether (a) closes the gap to (b), and whether (c) closes the
  gap to (a)/(b) given enough data (the bitter-lesson check).

## Compute budget

Small synthetic images/patterns, small models — well under a GPU-day, dominated by the
sweep over training-set sizes rather than model size.

## Bitter-lesson check

- Explicitly the highest-risk thread on this axis, and the experiment is designed to
  measure the risk directly (the plain-MLP-with-more-data control arm) rather than assume
  the architecture wins.
- If it survives, the win is "automating the discovery of general structure" rather than
  "encoding a general structure" — a legitimate use of computation, not a hand-coded
  shortcut, provided the plain-baseline control arm confirms it isn't just a a data-
  efficiency trick that scale erases.

## Known prior work / risk of reinventing

Related to prior work on learning symmetries / equivariance discovery (e.g. work on
"meta-learning symmetries," Lie-algebra-parameterized equivariant layers). The contribution
here is the explicit three-arm falsification protocol (learned vs. hand-built vs.
plain-with-scale), which most prior work doesn't run as a dedicated ablation.
